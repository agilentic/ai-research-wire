# AI Research Wire

AI Research Wire is a GitHub-native publishing site for AI research notes, news synthesis, paper-style essays, and research digests. It is inspired by the long-form research depth of [Gwern.net](https://gwern.net/) and the structured intelligence-publication model of [TuringWire](https://turingwire.com/).

## Architecture

- **GitHub is the source of truth.** All content and publishing logic is reviewable through branches and pull requests.
- **Markdown essays:** `research/posts/*.md`
- **Article metadata:** `data/articles.json`
- **Static output:** `public/`, generated by `scripts/build_site.py`
- **Hosting:** GitHub Pages through `.github/workflows/publish.yml`
- **Scheduled publishing:** GitHub Actions runs the build on a daily cron and on manual dispatch.

## Categories

The editorial taxonomy is:

- AI labs
- Model releases
- Agentic workflows
- Scientific papers
- Infra
- Safety
- Startups

## Local build

```bash
python scripts/build_site.py
python -m http.server 8000 -d public
```

Then open <http://localhost:8000>.

## Add a research essay manually

```bash
python scripts/publish_research.py \
  --title "Example Research Essay" \
  --category "scientific papers" \
  --summary "A short summary for the homepage." \
  --body-file /path/to/body.md

python scripts/build_site.py
```

The script writes:

- `research/posts/<slug>.md`
- a matching record in `data/articles.json`

## Hermes editorial workflow from Telegram

When Michael asks Hermes to research a topic in Telegram, Hermes should:

1. Research the topic using trusted sources and preserve citations in the Markdown body.
2. Create a new branch from the default branch.
3. Run `scripts/publish_research.py` with the researched title, category, summary, and Markdown body.
4. Run `python scripts/build_site.py`.
5. Run checks:
   - `python -m py_compile scripts/build_site.py scripts/publish_research.py`
   - `python scripts/build_site.py`
6. Rebase on the default branch.
7. Push the branch.
8. Open a **draft PR**.
9. Send the PR link back in Telegram.

## Repository rules

- Never push directly to `main` after initial bootstrap.
- Never force-push.
- Never delete the repository.
- Use PR-based versioning.
- Use GitHub Actions secrets for API keys.
- Keep generated, agent-authored research auditable through Markdown diffs.

## GitHub Pages setup

The workflow uses GitHub's official Pages artifact deployment actions. In repository settings, set Pages to deploy from **GitHub Actions**.

## API keys and automation

Do not commit API keys. If future scheduled research jobs call external APIs, store secrets in GitHub Actions secrets and read them from environment variables inside workflows.
