# Frontier Model Release Watch

A useful model-release digest should separate claims from evidence. Benchmarks, pricing pages, system cards, API documentation, context-window claims, and early developer reports should be tracked as separate signals rather than blended into one launch narrative.

## What to track

- Lab and model name
- Release date
- Modalities
- Context window and output limits
- Pricing and rate limits
- Tool-use support
- Safety notes and system card claims
- Benchmark claims
- Independent replication attempts
- Production adoption signals

## Editorial stance

The site should not simply repeat launch copy. It should highlight where a release changes what builders can do, what remains unverified, and what follow-up experiments Hermes or a human should run.
